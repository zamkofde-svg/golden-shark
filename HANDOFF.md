## Как публиковать изменения
1. Правишь файлы в `golden-shark`.
2. `git add -A && git commit -m "…" && git push` → Pages обновляется автоматически.
3. **Кэш:** при правке `assets/css/style.css` или `assets/js/*.js` поднимай версию `?v=N` в ссылках во всех `*.html`
   (сейчас css на `?v=6`, js на `?v=2`, logo/favicon на `?v=2`). Без бампа браузер держит старый файл.
   Команда: `find . -maxdepth 1 -name '*.html' -exec perl -i -pe 's{(style\.css\?v=)6}{${1}7}g;' {} +`

## ВАЖНО: бэкенд не работает на GitHub Pages
Pages раздаёт только статику — PHP (`api/`) не исполняется. Поэтому сейчас:
- Рейтинг и карточки турниров показываются из **демо-данных** (фолбэк в `main.js`/`account.js`).
- Запись на турнир, вход через Telegram, реальное расписание — **не работают**.

Чтобы заработало по-настоящему — нужен **PHP-хостинг** (как у «Зацепа» на reg.ru):
1. Залить весь проект на сервер с PHP+MySQL.
2. Создать `api/config.php` (по образцу `api/config.example.php`) с ключами Telegram-бота и доступом к БД.
3. Накатить `api/schema.sql` в phpMyAdmin.
4. Привязать домен (когда купят `goldenshark.fun`): добавить файл `CNAME` с доменом в корень репо + прописать DNS.
   og/canonical уже указывают на `goldenshark.fun`.
